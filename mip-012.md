# [MIP-012] Bubblegum Instruction to Close Empty Merkle Tree

[Submission details](https://github.com/metaplex-foundation/mip/discussions/50) & [proposed technical implementation](https://github.com/metaplex-foundation/mpl-bubblegum/pull/153) 

# Overview

The Bubblegum program currently does not include an instruction for closing an empty Merkle Tree. Adding an instruction to close empty Merkle Trees will allow creators to reclaim rent from Merkle Trees that are no longer needed.

# Motivation

Certain use cases for cNFTs would benefit from allowing the creator to close Merkle Trees that are no longer needed. For example, a market creator may choose to use a small Merkle Tree to start, but then later decide to expand the number of cNFTs they can offer. With an added instruction for closing an empty Merkle Trees, the creator could open a larger Merkle Tree account, migrate the existing cNFTs from the small tree to the larger tree, then close the empty tree to claim rent.

Similarly, if the use case calls for short lived ownership or access control, it is likely such cNFTs will eventually become worthless. The creator would benefit from being able to close the associated Merkle Tree to reclaim rent.

#Proposal

The proposal is to add a close_tree instruction to the Bubblegum program to close an empty Merkle Tree and its config account PDA, reclaiming rent.
